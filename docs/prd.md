# PRD

## 1. 项目名称
Frontend Interview Copilot

## 2. 项目简介
这是一个面向前端实习与校招准备场景的 AI 练习产品。它不再采用旧版“单输入页直接开始”的结构，而是围绕简历准备、分析、模拟问答和复盘沉淀，提供一条更清晰的工作流。

当前正式主链路为：

`/resume -> /analysis -> /interview -> /report`

## 3. 目标用户
- 正在准备前端实习或校招的学生
- 希望先梳理简历表达，再进行模拟面试练习的人
- 需要保留本地练习记录、便于回看的用户

## 4. 产品目标
- 让用户先准备简历和岗位信息，再进入分析与面试准备
- 用 AI 帮助用户理解简历优势、风险点和岗位匹配度
- 提供一轮轻量但完整的模拟问答、动态追问与复盘闭环
- 通过本地历史记录沉淀每次练习结果

## 5. 当前用户流程
1. 进入首页了解产品能力与入口
2. 在 `/resume` 输入或导入简历，并补充目标岗位 JD
3. 进入 `/analysis` 做简历分析、JD 匹配、简历聊天
4. 在 `/analysis` 发起模拟面试，进入 `/interview`
5. 在 `/interview` 逐题作答，并可能进入单题内的动态追问链
6. 进入 `/report` 生成并查看 AI 复盘报告
7. 在 `/history` 和 `/history/[id]` 回看本地历史记录

## 6. 页面职责
- `/`：首页与产品说明页，只负责介绍和导航
- `/resume`：输入页，负责准备简历草稿、目标岗位 JD，并支持 `.md` 简历导入
- `/analysis`：分析与面试准备页，负责简历分析、JD 匹配、简历聊天和开始模拟面试
- `/interview`：问答页，负责逐题作答、单题内动态追问与本地保存当前进度
- `/report`：复盘页，负责生成 AI 报告，并结合主问题与追问链进行复盘后写入本地历史
- `/history`：本地历史列表页
- `/history/[id]`：本地历史详情页，保留主问题与追问链的完整问答顺序
- `/setup`：兼容过渡入口，已退出正式主流程

## 7. 核心功能

### 7.1 简历与岗位信息准备
用户在 `/resume` 输入简历内容、输入岗位 JD，或导入 `.md` 简历文件。内容会自动保存在当前浏览器。

### 7.2 简历分析
用户在 `/analysis` 发起 AI 简历分析，获得总结、优势、风险点、改进建议和关键词。

### 7.3 JD 匹配
用户在 `/analysis` 基于当前简历与目标岗位 JD 生成岗位匹配分析。

### 7.4 简历聊天
用户在 `/analysis` 基于当前简历和分析结果继续追问，聊天用于辅助简历表达与准备，不替代正式面试问答页。

### 7.5 模拟问答
用户在 `/analysis` 开始模拟面试后进入 `/interview`，按顺序逐题回答问题，并保留本地进度。

当前问答行为定义：
- 主问题后可根据回答内容继续追问
- 单个主问题下最多 3 轮连续追问
- 0 / 1 / 2 / 3 轮都可能发生
- 第 3 轮后必须停止并进入下一主问题
- 追问链属于本轮正式问答，不是额外功能页
- 追问链结果会进入复盘与历史详情

### 7.6 复盘报告
用户在 `/report` 查看 AI 生成的评分、总结、优势、薄弱点和建议。报告会结合主问题及其后续追问链一起评估。

### 7.7 本地历史记录
用户可在 `/history` 和 `/history/[id]` 查看已保存到当前浏览器的历史快照，包括主问题与追问链的完整问答顺序。

## 8. 非功能要求
- 页面结构清晰，便于展示和投递
- 响应式布局
- 具备 loading / error / empty 状态
- 本地自动保存关键工作区状态
- 文案与产品结构保持一致，不混淆兼容逻辑和正式流程

## 9. 当前边界
- 不支持无限追问
- 不支持 PDF 上传与解析
- 不支持云同步
- 不支持导出报告
- 不支持多用户与登录体系
- 不支持语音面试
- 不提供全局聊天能力

补充说明：
- 当前支持 `.md` 简历导入
- `/setup` 仅用于兼容旧状态，不应作为正式页面能力对外描述
- 当前追问链不是树结构问答，也不是开放式长对话系统
