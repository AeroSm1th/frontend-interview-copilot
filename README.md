# Frontend Interview Copilot

一个面向前端求职准备的 AI 练习产品，围绕简历整理、岗位匹配、模拟问答、动态追问与复盘沉淀，提供一条小而完整的本地化练习链路。

## 项目简介
当前版本已经从旧版单输入页流程，演进为更清晰的多页面工作区结构。正式主链路为：

`/resume -> /analysis -> /interview -> /report`

在正式主链路之外，当前版本还支持一个轻量的专项强化训练入口：
- 用户可在 `/report` 基于当前报告中的 `weaknesses` 和 `suggestions` 发起“针对薄弱点再练”
- 系统会生成一组新的专项训练题，并进入新一轮 `/interview`
- 专项训练继续复用现有 `/interview -> /report -> /history` 链路
- `/history` 和 `/history/[id]` 会区分“普通模拟”和“专项训练”

问答阶段支持基于用户回答内容生成动态追问链：
- 单个主问题下最多支持 3 轮连续追问
- 0 / 1 / 2 / 3 轮都可能发生
- 第 3 轮后必须停止并进入下一主问题
- 追问链会进入 `/report` 与 `/history/[id]` 链路

其中：
- `/resume` 负责准备简历与岗位信息
- `/analysis` 负责简历分析、JD 匹配、简历聊天和开始面试
- `/interview` 负责逐题作答与动态追问
- `/report` 负责生成复盘报告
- `/history` 和 `/history/[id]` 负责查看本地历史记录
- `/setup` 仅作为兼容过渡入口，已退出正式主流程

## 当前核心功能
- 简历文本输入与本地自动保存
- `.md` 简历导入
- 目标岗位 JD 输入与本地自动保存
- AI 简历分析
- AI JD 匹配分析
- 基于简历分析结果的简历聊天
- AI 生成面试题并进入问答流程
- 动态追问链：单个主问题下最多 3 轮连续追问
- AI 生成复盘报告
- 基于复盘报告薄弱点的专项强化训练
- 本地历史记录列表与详情页

## 主要页面说明

### `/`
首页与产品介绍页，只负责说明产品能力和提供入口，不承载业务流程状态。

### `/resume`
输入页。用于整理 source material，包括简历草稿、目标岗位 JD，以及 `.md` 简历导入。

### `/analysis`
分析与面试准备页。读取 `/resume` 阶段保存的内容，完成：
- 简历分析
- JD 匹配
- 简历聊天
- 开始模拟面试

聊天侧边栏只在 `/analysis` 激活。

### `/interview`
问答页。承接分析阶段准备好的上下文与题目，或承接 `/report` 发起的一轮专项训练题，按顺序逐题作答，并根据当前回答决定是否继续追加追问。

当前规则：
- 单个主问题下最多 3 轮连续追问
- 0 / 1 / 2 / 3 轮都可能发生
- 第 3 轮后必须停止并进入下一主问题
- 当前进度会持续保存在本地

### `/report`
复盘页。基于当前整轮问答结果生成 AI 报告，并在成功生成后写入本地历史记录。报告会一并评估主问题及其后续追问链表现。普通模拟报告页还可以继续发起一轮“针对薄弱点再练”专项训练。

### `/history`
本地历史列表页，展示当前浏览器内保存的历史快照，并区分普通模拟与专项训练。

### `/history/[id]`
本地历史详情页，查看单次练习的报告、题目、回答、JD 和简历快照。题目明细会保留主问题及其后续追问链的原始顺序；如果是专项训练，也会展示本轮聚焦的薄弱点和建议。

### `/setup`
兼容过渡入口，用于承接旧状态并跳转到 `/analysis`。它不是当前正式主流程的一部分。

## 项目流程说明
1. 用户在 `/resume` 输入或导入简历，并补充目标岗位 JD
2. 进入 `/analysis`，发起 AI 简历分析
3. 视需要继续做 JD 匹配和简历聊天
4. 在 `/analysis` 发起模拟面试，生成题目并进入 `/interview`
5. 在 `/interview` 逐题作答；每个主问题后可能触发 0 到 3 轮连续动态追问，第 3 轮后停止
6. 进入 `/report` 生成 AI 复盘报告，报告会结合主问题与追问链一起评估
7. 报告成功生成后写入 `/history`，供后续查看与回放完整问答链
8. 如果需要，用户可在 `/report` 基于薄弱点继续发起一轮专项强化训练，并复用现有 `/interview -> /report -> /history` 链路

## 本地状态与数据保存
当前版本使用浏览器本地存储保存工作区状态，不依赖登录或云端服务。主要包括：
- 简历草稿与 JD 草稿
- 简历分析与 JD 匹配缓存
- 简历聊天记录
- 当前面试会话
- 当前复盘报告
- 历史记录快照

其中面试相关状态会区分：
- 普通模拟
- 专项训练

这意味着：
- 数据只保存在当前浏览器
- 更换浏览器或清空本地数据后不会保留
- 当前不支持跨设备同步

## 环境变量
当前服务端接口通过阿里云百炼兼容接口调用模型，环境变量建议在 `.env.local` 中配置：

```bash
DASHSCOPE_API_KEY=your_dashscope_api_key
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL=qwen-turbo
```

说明：
- `DASHSCOPE_API_KEY`：必填，用于服务端调用模型
- `DASHSCOPE_BASE_URL`：可选，默认使用百炼兼容接口地址
- `AI_MODEL`：可选，默认值为 `qwen-turbo`

## 当前边界说明
当前版本重点展示一条完整但克制的前端求职练习链路，以下能力暂未实现：
- 无限追问
- 树结构问答
- PDF 上传与解析
- 云同步
- 多用户与账号体系
- 导出报告
- 语音面试
- 全局聊天入口
- 跨页面通用聊天侧边栏

补充说明：
- 当前支持的是 `.md` 简历导入，不是通用文件解析
- `/setup` 是兼容过渡入口，不是正式输入页
- 文档与展示口径应以 `/resume -> /analysis -> /interview -> /report` 为准
- 当前动态追问链只支持单个主问题下最多 3 轮连续追问，不是开放式深挖系统
